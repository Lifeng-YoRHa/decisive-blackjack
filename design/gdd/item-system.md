# 道具系统 (Item System)

> **Status**: In Design
> **Author**: user + agents
> **Last Updated**: 2026-04-26
> **Implements Pillar**: Gameplay — tactical items during card sorting phase

## Overview

道具系统是《决胜21点》的战术消耗品机制。玩家在商店中购买最多 5 件道具，在对战回合的排序阶段（Phase 6）使用，产生即时战斗效果（回复、伤害）、信息优势（窥视牌堆、揭示暗牌）、手牌操控（移除手牌的品质与印记，牌回抽牌堆）、排序干扰（锁定对手卡牌位置）或延迟防御（结算前获得额外防御）。所有道具单次使用后消耗。道具为玩家专属机制，AI 不使用道具。

在数据层面，道具系统维护一个玩家道具库存（最多 5 件的 ItemInstance 列表），通过筹码经济系统执行购买扣款，通过战斗状态系统执行即时回复/伤害，通过卡牌排序系统集成密码锁的位置锁定，通过卡牌数据模型处理微缩炸药的品质/印记移除。道具的生效时机分为三类：即时（使用即生效）、排序修饰（影响手动排序阶段，在印记自动排序前释放）、延迟（效果排队至结算前触发）。在体验层面，道具为排序阶段增加了第二个决策维度——玩家不再只决定卡牌顺序，还要决定是否在此时消耗一件珍贵的道具来改变战局。

## Player Fantasy

**百宝箱赌徒 — 暗藏王牌，关键时刻翻盘**

核心时刻：排序阶段。你手里有 5 张牌，对手明牌是方片 Q（12 伤害）。你 HP 只剩 18。你打开道具栏——透视眼镜让你看到自己牌堆顶 3 张全是低点数，接下来几回合不会爆牌，但这一回合你必须在排序中活下来。你用能量饮料回复 10 HP（28 了），再对小刀对手造成 10 伤害先削弱他。然后你给对手那张方片 Q 上密码锁——它被锁在当前位置，AI 不能把它挪到对你更不利的位置。你确认排序，锁解除，结算开始。28 HP，10 防御（厚衣服），对手方片 Q 打过来被防御吃掉大部分。你活下来了。5 件道具用了 3 件，剩 2 件留给后面更难的对手。

道具系统的玩家幻想是**赌桌上的百宝箱赌徒**：你不是赤手空拳上桌——你带着工具。每件道具都是一次单发的战术选择：透视眼镜让你"偷看命运"（信息不对称），小镜子让你"识破对手"（暗牌变明牌），微缩炸药让你"壮士断腕"（牺牲一张牌的品质来降低点数避免爆牌），密码锁让你"绑住对手的手"（限制对方排序），厚衣服是"防弹衣"（提前攒防御），能量饮料和小刀是简单的回血和伤害。道具的永久保留特性让你可以做"存道具"的长期规划——这一回合不用，留给 Boss 战。而 5 件上限迫使你做取舍：买新道具还是省着筹码买印记？留着旧道具还是用掉腾出空位？

这个幻想与商店系统的"军需官"和印记系统的"锻造师"一脉相承：商店说你分配的是战争预算，印记说你锻造的是时序编排，道具说你随身携带的是战术锦囊。三者共享同一笔筹码——买印记还是买道具，是构筑层面（印记）和战术层面（道具）的取舍。印记是永久的建筑决策，道具是单发的战术弹药。

## Detailed Design

### Core Rules

**1. 道具类型定义**

7 种道具，分为 3 个生效时机类别：

| 道具 | 枚举值 | 类别 | 效果 | 售价 |
|------|--------|------|------|------|
| 能量饮料 | `ENERGY_DRINK` | 即时 | 回复 10 HP | 70 |
| 小刀 | `KNIFE` | 即时 | 对 AI 造成 10 伤害 | 70 |
| 透视眼镜 | `XRAY_GLASSES` | 即时 | 查看自己抽牌堆顶 3 张 | 150 |
| 小镜子 | `SMALL_MIRROR` | 即时 | 揭示对手一张暗牌（玩家选择） | 100 |
| 微缩炸药 | `MINI_EXPLOSIVE` | 即时 | 移除手牌的品质与印记，牌回抽牌堆 | 150 |
| 密码锁 | `PADLOCK` | 排序修饰 | 锁定对手一张牌位置（仅 Pass 1） | 100 |
| 厚衣服 | `THICK_CLOTHES` | 延迟 | 第一张卡牌结算前 +10 防御 | 60 |

**2. 道具库存**

| 字段 | 类型 | 说明 |
|------|------|------|
| `inventory` | Array[ItemInstance] | 玩家持有的道具列表 |
| `max_items` | int | 同时携带上限 = 5 |

- 同种道具可携带多个（如 3 瓶能量饮料）
- 道具一经购买即永久保留，直到被使用
- 使用后从 `inventory` 中移除

**3. 购买规则**

- 道具在商店系统中购买，属于固定服务类别（不限次、不刷新）
- 购买条件：`len(inventory) < max_items` 且筹码足够
- 购买时调用 `spend_chips(PLAYER, item_price, SHOP_PURCHASE)`
- 每个新道具占 1 个库存位。库存满时不允许购买新道具

**4. 使用时机**

- 道具仅能在排序阶段（Phase 6）使用
- 排序阶段内，道具使用与拖拽排序可自由交替（无先后顺序要求）
- 玩家确认排序后，道具使用窗口关闭（不可再使用任何道具）
- 单次排序阶段可使用任意数量的道具（受限于库存）
- 道具使用消耗排序计时器时间（默认 30 秒）。每次道具交互（选择→确认）预计 3-5 秒。玩家需在计时器内同时完成排序和道具使用。所有道具使用不可撤销——尤其是微缩炸药，误用无回退

**5. 即时类道具规则**

即时类道具在使用的瞬间立即生效，效果不可撤销：

- **能量饮料**：调用战斗状态系统的 `apply_heal(PLAYER, 10)`。受 `max_hp` 截断规则约束（满 HP 时回复浪费）。与红桃回复使用同一接口。
- **小刀**：调用战斗状态系统的 `apply_damage(AI, 10)`。此时双方 `defense = 0`（每回合开始已重置），因此 10 伤害全额扣 HP。AI HP 最低为 0。
- **透视眼镜**：读取玩家抽牌堆顶部最多 3 张牌的完整信息（suit, rank, stamp, quality）。如果抽牌堆不足 3 张，显示全部可用牌。信息仅展示给玩家（UI 渲染），不修改任何游戏状态。使用后玩家在剩余对局中可参考此信息（记忆由玩家自己负责）。
- **小镜子**：玩家点击选择对手的一张暗牌，系统揭示该牌的完整信息（suit, rank, stamp, quality）。信息仅展示给玩家。对手的暗牌数量 = 对手手牌数 - 1（第一张为明牌）。
- **微缩炸药**：玩家选择自己的一张手牌。该牌的 `quality` 设为 `null`，`stamp` 设为 `null`（卡质和印记永久移除，不可恢复）。该牌从手牌中移除，放回玩家抽牌堆的底部。手牌数减少 1。**注意**：此操作不影响牌组的 52 张不变量——牌只是从手牌回到抽牌堆，且品质/印记被清除。
- **微缩炸药核心用途**：移除手牌中的一张牌，减少手牌数量、改变花色组成。代价是永久失去该牌的品质和印记投资，以及手牌数减少（注：牌型倍率在 Phase 5 已锁定，移除卡牌不改变已锁定的倍率——存活的卡牌仍按原倍率结算）。**注意**：爆牌检测使用点数计算引擎在 Phase 5 结束时冻结的 `point_result`——排序阶段（Phase 6）移除卡牌不会重新计算 `point_result.is_bust`，因此微缩炸药不能改变爆牌状态。这是一次刻意的经济牺牲——用投资换取手牌操控（减少结算牌数、移除不利花色）。

**6. 排序修饰类道具规则**

- **密码锁**：玩家选择对手的一张手牌（明牌或暗牌均可，按卡牌实例锁定而非位置）。被选中的卡牌实例在排序阶段 Pass 1（手动排序 / AI tiebreak）期间被锁定——该卡牌不可被移动到其他位置。锁定在 Pass 1 结束时（玩家确认排序后）立即解除，然后 Pass 2（印记自动排序）正常执行。这意味着：
  - 锁定仅阻止手动位置调整，不阻止印记驱动的自动排序
  - 被锁定的跑鞋卡牌在 Pass 1 期间停在原位，解锁后 Pass 2 将其推至最前
  - 被锁定的乌龟卡牌在 Pass 1 期间停在原位，解锁后 Pass 2 将其推至最后
  - 被锁定的无特殊印记卡牌在解锁后位置不变（Pass 2 不影响默认组）

**7. 延迟类道具规则**

- **厚衣服**：使用时道具从库存消耗，但效果排队等待。在排序阶段完全结束（双方排序确认 + 印记自动排序完成）后、第一张卡牌结算前，调用 `add_defense(PLAYER, 10)`。此防御叠加在已有防御之上（回合开始时 defense=0，厚衣服是最早添加的防御）。后续的黑桃卡牌结算继续正常累积防御。`pending_defense` 状态存储在战斗状态系统中——道具系统调用 `queue_defense(PLAYER, amount)` 排队，战斗状态系统在"排序结束→结算开始"的间隙执行所有排队的防御。多件厚衣服按使用顺序依次触发（FIFO）。

**8. 道具使用与排序阶段的交互时序**

```
排序阶段激活（初始顺序 = 发牌顺序）
       │
       ├── 玩家自由操作：
       │   ├── 拖拽重排手牌（可无限次撤销）
       │   ├── 使用即时类道具（能量饮料、小刀、透视眼镜、小镜子、微缩炸药）
       │   ├── 使用密码锁（锁定对手一张牌）
       │   └── 使用厚衣服（效果排队）
       │   注：以上操作可自由交替，无固定顺序
       │
       ▼
玩家确认排序 → manual_order 锁定 → 道具使用窗口关闭
       │
       ▼
密码锁解除 → 锁定卡牌恢复可移动状态
       │
       ▼
印记自动排序（Pass 2）→ stable_sort_by(stamp_sort_key, manual_order)
       │
       ▼
厚衣服效果触发 → add_defense(PLAYER, 10)
       │
       ▼
最终排序展示 → pos 1, 2, 3... 编号冻结
       │
       ▼
结算开始
```

**9. 道具数据模型**

```
ItemInstance:
  item_type: ItemType          # 枚举（7 种）
  purchase_price: int          # 购买价格（用于卖出/统计，不影响玩法）
  purchase_round: int          # 购买时的对手序号（用于统计）
```

**10. 跨回合和跨对手保留**

- 道具库存跨回合保留（同一对手的不同回合之间不重置）
- 道具库存跨对手保留（击败对手后不重置）
- 道具库存跨商店保留（商店购物期间可查看库存状态，但不可使用道具——使用仅限排序阶段）
- 仅在游戏结束时（VICTORY / GAME_OVER）清空

### States and Transitions

```
[商店阶段] ─── 购买道具 → 加入 inventory
       │
       ▼
[对战回合 → 排序阶段激活]
       │
       ├── inventory 非空 → 显示可用道具栏
       │   │
       │   ├── 玩家点击道具 → 选择目标（如适用）→ 确认
       │   │   │
       │   │   ├── 即时类 → 效果立即应用
       │   │   ├── 密码锁 → 标记目标卡牌为 locked
       │   │   └── 厚衣服 → 标记 pending_defense += 10
       │   │
       │   └── 道具从 inventory 移除
       │
       ▼
[玩家确认排序]
       │
       ├── 解除所有密码锁
       ├── 执行 Pass 2（印记自动排序）
       ├── 执行 pending_defense（厚衣服效果）
       │
       ▼
[结算开始] → 道具效果已全部应用
```

## Formulas

### 1. 道具定价 (item_price_lookup)

道具价格通过查找表确定，无公式计算：

```
item_price = lookup(item_price_table, item_type)
```

| item_type | item_price |
|-----------|-----------|
| `ENERGY_DRINK` | 70 |
| `XRAY_GLASSES` | 150 |
| `SMALL_MIRROR` | 100 |
| `MINI_EXPLOSIVE` | 150 |
| `KNIFE` | 70 |
| `THICK_CLOTHES` | 60 |
| `PADLOCK` | 100 |

**变量：**
| 变量 | 类型 | 范围 | 说明 |
|------|------|------|------|
| item_type | enum | 7 种 | 道具类型枚举 |
| item_price | int | 60 ~ 150 | 道具售价（筹码） |

**定价参考**：与商店已有服务对比
- 能量饮料 70 筹码 / 10 HP = 7.0 筹码/HP（商店 HP 恢复为 5 筹码/HP，道具贵 40%）
- 小刀 70 筹码 / 10 伤害 = 7.0 筹码/伤害
- 厚衣服 60 筹码 / 10 防御 = 6.0 筹码/防御
- 信息类（透视眼镜 150、小镜子 100）价值取决于玩家利用信息的能力，无法直接对比

### 2. 能量饮料回复 (energy_drink_heal)

```
heal_amount = 10
hp_new = min(max_hp, hp + heal_amount)
overflow = max(0, hp + heal_amount - max_hp)
```

直接调用战斗状态系统的 `apply_heal(PLAYER, 10)`，遵循现有回复规则（截断 max_hp，溢出浪费）。

**变量：**
| 变量 | 类型 | 范围 | 说明 |
|------|------|------|------|
| heal_amount | int | 固定 10 | 能量饮料回复量（调参点 `item_energy_drink_heal`） |
| hp | int | 0 ~ max_hp | 玩家当前 HP |
| max_hp | int | 100 | 玩家 HP 上限 |

**示例**：hp=85, max_hp=100 → hp_new=95, overflow=0。hp=95, max_hp=100 → hp_new=100, overflow=5。

### 3. 小刀伤害 (knife_damage)

```
damage_amount = 10
hp_new_ai = max(0, ai_hp - damage_amount)
```

直接调用战斗状态系统的 `apply_damage(AI, 10)`。排序阶段 AI defense=0（回合开始时已重置），因此伤害全额扣 HP。

**变量：**
| 变量 | 类型 | 范围 | 说明 |
|------|------|------|------|
| damage_amount | int | 固定 10 | 小刀伤害值（调参点 `item_knife_damage`） |
| ai_hp | int | 0 ~ ai_max_hp | AI 当前 HP |

**示例**：ai_hp=80, damage=10 → ai_hp_new=70。ai_hp=5, damage=10 → ai_hp_new=0。

### 4. 透视眼镜揭示数 (xray_reveal_count)

```
reveal_count = min(3, draw_pile_size)
revealed_cards = draw_pile[0:reveal_count]
```

**变量：**
| 变量 | 类型 | 范围 | 说明 |
|------|------|------|------|
| reveal_count | int | 固定 3（上限） | 揭示牌数（调参点 `item_xray_reveal_count`） |
| draw_pile_size | int | 0 ~ 52 | 玩家抽牌堆剩余牌数 |

**示例**：draw_pile 有 30 张 → 揭示前 3 张。draw_pile 有 1 张 → 揭示 1 张。draw_pile 为空 → 揭示 0 张（信息为空，道具仍消耗）。

### 5. 厚衣服防御 (thick_clothes_defense)

```
defense_bonus = 10
defense_new = defense + defense_bonus  // 在结算前、所有卡牌结算前应用
```

调用战斗状态系统的 `queue_defense(PLAYER, 10)`（由战斗状态系统在排序结束后 FIFO 执行 `add_defense`）。时机：排序完全结束后、第一张卡牌结算前。此防御与后续黑桃卡牌结算累积的防御叠加（defense 无上限）。

**变量：**
| 变量 | 类型 | 范围 | 说明 |
|------|------|------|------|
| defense_bonus | int | 固定 10 | 厚衣服防御值（调参点 `item_thick_clothes_defense`） |
| defense | int | 0 ~ defense_bonus × n_thick_clothes | 应用前的防御值。首件触发时为 0，多件叠加时逐次递增 |

**示例**：回合开始 defense=0，厚衣服 +10 → defense=10。随后黑桃 K 结算 → defense=23。然后方片 Q(12伤害) 结算 → absorbed=12, defense_new=11, hp 不变。

### 6. 库存容量检查 (inventory_capacity_check)

```
can_purchase = len(inventory) < max_items AND chip_balance >= item_price
```

**变量：**
| 变量 | 类型 | 范围 | 说明 |
|------|------|------|------|
| inventory | Array[ItemInstance] | 0 ~ 5 | 当前道具库存 |
| max_items | int | 固定 5 | 最大携带数（调参点 `item_max_inventory`） |
| chip_balance | int | 0 ~ ∞ | 玩家当前筹码余额 |
| item_price | int | 60 ~ 150 | 道具售价 |

**示例**：inventory 有 3 件，购买能量饮料(70)，筹码余额 200 → can_purchase=true。inventory 有 5 件 → can_purchase=false（库存满）。

## Edge Cases

- **如果能量饮料使用时 HP 已满**：回复值全部浪费（overflow），道具仍消耗。调用 `apply_heal(PLAYER, 10)` 后 hp 不变，overflow=10。玩家应避免满 HP 时使用能量饮料——UI 应显示当前 HP 提醒。

- **如果小刀将 AI HP 降至 0 或以下**：AI HP 截断为 0，道具正常消耗。但这**不触发生死判定**——生死判定仅在所有卡牌结算完毕且防御清零后统一执行（遵循战斗状态系统规则 9）。AI 可能在后续卡牌结算中被红桃回复救回。小刀只是提前削减了 AI 的 HP。

- **如果透视眼镜使用时抽牌堆为空**：reveal_count=0，不揭示任何牌，道具仍消耗。这是玩家的失误——使用前应评估抽牌堆状态。52 张不变量保证在弃牌堆重洗后抽牌堆不会长期为空，但在洗牌过渡的瞬间可能出现。

- **如果透视眼镜使用时抽牌堆只有 1-2 张**：揭示全部可用牌（1 或 2 张），不补足 3 张。道具仍消耗。UI 应清晰展示揭示的牌数，避免玩家误以为看到了 3 张。

- **如果小镜子使用时对手没有暗牌**：不应发生——排序阶段在发牌之后，对手至少有 1 张暗牌（2 张手牌中 1 张明牌）。如果因特殊玩法（分牌等）导致对手只有 1 张牌且为明牌，小镜子不可使用（UI 禁用）。

- **如果微缩炸药使用在无品质无印记的牌上**：品质和印记已经是 null，操作为 no-op（设置 null → null）。牌仍从手牌移除并放回抽牌堆底部，手牌数减少 1。道具仍消耗。这不是最优使用方式，但规则允许。

- **如果微缩炸药使用在最后一张手牌上**：手牌变为空。空手牌不参与结算（排序系统返回空列表，结算引擎跳过该方）。该牌回到抽牌堆底部。这在极端情况下是有意为之——牺牲全部手牌来减少结算牌数，但代价是本回合零输出。**注意**：微缩炸药的使用时机在排序阶段（Phase 6），此时点数计算引擎已在 Phase 5 结束时冻结 `point_result`——使用微缩炸药不会重新计算 `point_result.is_bust`，因此不能改变爆牌状态。移除手牌仅影响手牌的花色组成和数量。

- **如果密码锁用在跑鞋卡牌上**：Pass 1 期间该牌被锁定在当前位置，AI 无法手动移动它。Pass 1 结束后锁解除，Pass 2 中跑鞋印记仍将该牌推至最前组。玩家需要意识到锁定跑鞋牌只是延迟了它的自动移动——如果目的是阻止跑鞋牌抢先结算，密码锁无法达成此目的。

- **如果密码锁用在乌龟卡牌上**：同理，Pass 1 锁定，解锁后 Pass 2 将乌龟牌推至最后。锁定乌龟牌只阻止手动前移，不阻止自动后推。

- **如果同一对手卡牌被多个密码锁锁定**：冗余，效果不叠加。该牌仍然只是被锁定——多个密码锁等于一个密码锁。多出的密码锁浪费。

- **如果使用多件厚衣服**：每件独立消耗，防御叠加。使用 2 件厚衣服 → 结算前 `add_defense(PLAYER, 10)` 调用两次 → defense=20。每件占 1 个库存位，总投入 120 筹码获得 20 防御。

- **如果同一回合使用能量饮料 + 厚衣服**：两个效果独立生效。能量饮料立即回复 10 HP（即时类），厚衣服在结算前 +10 防御（延迟类）。时序不冲突。

- **如果同一回合使用小刀 + 密码锁**：小刀立即造成 10 伤害，密码锁锁定对手一张牌。两个效果独立生效，时序不冲突。小刀是即时类，密码锁是排序修饰类。

- **如果在分牌情况下使用道具**：道具作用于当前正在排序的手牌。密码锁锁定的是对手当前手牌中的卡牌。微缩炸药移除的是当前手牌中的卡牌。分牌的每手牌独立排序，但道具库存是共享的——在一手排序中用了道具，另一手排序时该道具已不在库存中。

- **如果道具库存满时进入商店**：无法购买新道具，但可以先用掉已有道具（在排序阶段使用后再回到商店——但排序阶段在商店之前，所以这种情况不会发生）。实际场景：商店阶段只能购买，不能使用。需要在后续对战中先用掉道具才能腾出空位。

- **如果玩家使用厚衣服后爆牌**：厚衣服防御在"排序完全结束后、第一张卡牌结算前"触发。爆牌检测在结算引擎的 Phase 0b 执行，即厚衣服防御之后。爆牌自伤绕过防御（`apply_bust_damage` 直接扣 HP）。但 AI 的卡牌仍正常结算——厚衣服的 10 防御会吸收 AI 的方片伤害。因此爆牌时厚衣服的防御并非完全浪费，只是无法抵消爆牌自伤本身。

- **如果微缩炸药导致牌型判定改变**：移除一张手牌可能使牌型检测的结果改变（如 FLUSH ×5 变为 ×4，或 PAIR 消失）。牌型检测在排序阶段之前（Phase 5 结束后、Phase 6 开始前）已执行完毕——但微缩炸药在排序阶段使用，此时牌型已锁定。需确认：牌型检测是在微缩炸药之前还是之后执行？如果之前，微缩炸药不影响当前回合的牌型；如果之后，需要重新检测。**设计决策**：牌型检测在 Phase 5 结束时锁定，微缩炸药在 Phase 6 使用时不触发重新检测。移除手牌不影响已锁定的牌型倍率——这是刻意的简化，避免牌型检测与道具系统的时序耦合。
- **如果微缩炸药移除 PAIR 中的一张牌**：牌型倍率在 Phase 5 锁定时已确定（如 PAIR ×2）。微缩炸药在 Phase 6 移除其中一张牌后，锁定倍率仍应用于存活的另一张牌（不重新检测，倍率不回退）。结算引擎在遍历排序列表时，仅对存在于手牌中的卡牌应用锁定倍率——被移除的牌已在抽牌堆中，不参与结算。同理，FLUSH ×5 移除一张后，剩余 4 张仍按 ×5 倍率结算。

| 调参点 | 类型 | 默认值 | 安全范围 | 影响什么 |
|--------|------|--------|----------|---------|

| `item_energy_drink_heal` | int | 10 | 5 ~ 30 | 能量饮料回复量。影响道具性价比（当前 7.0 筹码/HP vs 商店 5.0 筹码/HP） |

| `item_knife_price` | int | 70 | 30 ~ 200 | 小刀售价 |
| `item_xray_price` | int | 150 | 50 ~ 400 | 透视眼镜售价 |
| `item_mirror_price` | int | 100 | 50 ~ 300 | 小镜子售价 |
| `item_explosive_price` | int | 150 | 50 ~ 400 | 微缩炸药售价 |
| `item_thick_clothes_price` | int | 60 | 30 ~ 200 | 厚衣服售价 |
| `item_padlock_price` | int | 100 | 50 ~ 300 | 密码锁售价 |

**平衡备注**：
- 能量饮料（70/10HP）比商店 HP 恢复（50/10HP）贵 40%，溢价来自"战斗中即时使用"的灵活性
- 小刀（70/10伤害）等价于能量饮料——攻防对称定价
- 厚衣服（60/10防御）是最便宜的战斗道具——防御是回合末清零的临时资源，价值低于 HP
- 透视眼镜和微缩炸药（各 150）是最贵的——信息优势和手牌操控的战略价值最高
- 所有价格需要在实际游戏中验证经济平衡（筹码收入 vs 支出）

## Acceptance Criteria

### 购买与库存

**AC-01: 道具购买加入库存**
GIVEN 商店阶段，玩家筹码余额 200，inventory 有 3 件道具
WHEN 玩家购买能量饮料（售价 70）
THEN inventory 变为 4 件，筹码余额变为 130，新增 `ItemInstance(item_type=ENERGY_DRINK, purchase_price=70)`

**AC-02: 库存满时不可购买**
GIVEN 商店阶段，inventory 有 5 件道具（max_items=5）
WHEN 玩家尝试购买任何道具
THEN 购买被拒绝，inventory 仍为 5 件，筹码不变

**AC-03: 筹码不足时不可购买**
GIVEN 商店阶段，玩家筹码余额 50
WHEN 玩家尝试购买透视眼镜（售价 150）
THEN 购买被拒绝，筹码仍为 50

**AC-04: 同种道具可叠加携带**
GIVEN 商店阶段，inventory 已有 2 件能量饮料
WHEN 玩家再购买 1 件能量饮料
THEN inventory 变为 3 件能量饮料。同种道具各自独立，使用时消耗 1 件

### 即时类道具

**AC-05: 能量饮料回复 HP**
GIVEN 排序阶段，player hp=75, max_hp=100
WHEN 玩家使用能量饮料
THEN player.hp=85，道具从 inventory 移除

**AC-06: 能量饮料满 HP 溢出浪费**
GIVEN 排序阶段，player hp=95, max_hp=100
WHEN 玩家使用能量饮料
THEN player.hp=100（截断），overflow=5，道具仍消耗

**AC-07: 小刀对 AI 造成伤害**
GIVEN 排序阶段，ai hp=80, ai defense=0
WHEN 玩家使用小刀
THEN ai.hp=70，道具从 inventory 移除

**AC-08: 小刀将 AI HP 降至 0**
GIVEN 排序阶段，ai hp=8
WHEN 玩家使用小刀
THEN ai.hp=0（截断）。生死判定不在此时触发——等结算完毕后统一判定

**AC-09: 透视眼镜揭示牌堆顶 3 张**
GIVEN 排序阶段，玩家抽牌堆有 30 张牌，顶部为 [红桃K, 黑桃3+护盾, 方片J+红宝石II]
WHEN 玩家使用透视眼镜
THEN UI 展示这 3 张牌的完整信息（suit, rank, stamp, quality），道具消耗

**AC-10: 透视眼镜牌堆不足 3 张**
GIVEN 排序阶段，玩家抽牌堆只有 1 张牌 [草花7]
WHEN 玩家使用透视眼镜
THEN UI 展示 1 张牌的完整信息，道具仍消耗

**AC-11: 小镜子揭示对手暗牌**
GIVEN 排序阶段，对手手牌 3 张：明牌=方片Q，暗牌=[黑桃K, 红桃7+心]
WHEN 玩家使用小镜子并选择第二张暗牌（红桃7+心）
THEN UI 展示该牌的完整信息（suit=HEARTS, rank=7, stamp=HEART, quality=null），道具消耗

### 微缩炸药

**AC-12: 微缩炸药移除品质和印记**
GIVEN 排序阶段，玩家手牌包含方片9 + 红宝石III + 短剑印记
WHEN 玩家对此牌使用微缩炸药
THEN 该牌 quality=null, stamp=null，从手牌移除，放入抽牌堆底部。手牌数减少 1。道具消耗

**AC-13: 微缩炸药对无品质无印记的牌**
GIVEN 排序阶段，玩家手牌包含基础方片9（quality=null, stamp=null）
WHEN 玩家对此牌使用微缩炸药
THEN 该牌从手牌移除，放入抽牌堆底部。品质和印记不变（null→null）。手牌数减少 1。道具消耗

**AC-14: 微缩炸药用在最后一张手牌上**
GIVEN 排序阶段，玩家手牌仅 1 张（草花5）
WHEN 玩家对此牌使用微缩炸药
THEN 手牌变为空（[]），该牌回到抽牌堆。排序系统收到空手牌 → 跳过排序 → 无卡牌结算

### 密码锁

**AC-15: 密码锁阻止 AI 手动排序**
GIVEN 排序阶段，AI 手牌 [红桃J(pos1), 方片7+跑鞋(pos2), 黑桃K(pos3)]，无密码锁时 AI tiebreak 会将黑桃K移至 pos1
WHEN 玩家使用密码锁锁定黑桃K
THEN Pass 1 期间黑桃K 不能被移动。AI tiebreak 只能在红桃J 和方片7+跑鞋之间调整

**AC-16: 密码锁解锁后跑鞋正常排序**
GIVEN 密码锁锁定了对手的方片7+跑鞋（在 pos2）
WHEN 玩家确认排序（Pass 1 结束）→ 密码锁解除 → Pass 2 执行
THEN 方片7+跑鞋被印记自动排序推至最前组。锁定只阻止了手动调整，不阻止跑鞋的自动前移

**AC-17: 多个密码锁冗余**
GIVEN 玩家使用 2 个密码锁锁定对手同一张牌
THEN 效果不叠加，该牌仍只是被锁定。多出的密码锁浪费

### 厚衣服

**AC-18: 厚衣服在结算前添加防御**
GIVEN 排序阶段，player defense=0
WHEN 玩家使用厚衣服，确认排序，排序完成
THEN 排序完成后、第一张卡牌结算前，player.defense=10

**AC-19: 多件厚衣服防御叠加**
GIVEN 排序阶段，player defense=0，inventory 有 2 件厚衣服
WHEN 玩家使用 2 件厚衣服
THEN 结算前 add_defense(PLAYER, 10) 调用两次 → player.defense=20

**AC-20: 厚衣服防御与黑桃防御叠加**
GIVEN 排序阶段，player defense=0，玩家使用 1 件厚衣服，手牌中有黑桃K
WHEN 排序完成后厚衣服 +10 防御，随后黑桃K 结算
THEN 厚衣服先生效 defense=10，黑桃K 结算后 defense=23（10+13）

### 综合场景

**AC-21: 同一回合使用多种道具**
GIVEN 排序阶段，inventory 有 [能量饮料, 小刀, 厚衣服]，player hp=50, ai hp=80
WHEN 玩家依次使用能量饮料、小刀、厚衣服
THEN player.hp=60（能量饮料），ai.hp=70（小刀），inventory 变为空，厚衣服防御排队等待结算前触发

**AC-22: 确认排序后不可使用道具**
GIVEN 排序阶段，inventory 有 1 件能量饮料
WHEN 玩家确认排序
THEN 道具使用窗口关闭，能量饮料不可使用（保留在库存中，留到下回合）

**AC-23: 道具跨对手保留**
GIVEN 击败对手 3 时 inventory 有 [能量饮料, 小刀, 透视眼镜]
WHEN 进入对手 4 的战斗
THEN inventory 仍为 [能量饮料, 小刀, 透视眼镜]，3 件道具不变
