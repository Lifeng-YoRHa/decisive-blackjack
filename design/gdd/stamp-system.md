# 印记系统 (Stamp System)

> **Status**: In Design
> **Author**: user + agents
> **Last Updated**: 2026-04-24
> **Implements Pillar**: Core — card stamp effects and resolution pipeline integration

## Overview

印记系统是《决胜21点》的卡牌增强机制之一。它定义了 7 种印记类型（短剑、护盾、心、钱币、重锤、跑鞋、乌龟），每种赋予卡牌一个固定的结算效果——额外伤害、额外防御、额外回复、额外筹码、摧毁对手同位卡牌、优先结算、延后结算。印记通过商店购买附加到玩家的卡牌实例上，是游戏构筑系统的核心决策维度之一：在有限的筹码下选择给哪张牌打上哪种印记，直接决定了每回合结算的输出构成。

在数据层面，印记存储在 CardInstance 的 `stamp` 字段（可空枚举），由卡牌数据模型定义结构，由商店系统写入，由结算引擎在"弹出数值 → **印记效果** → 卡质效果 → 牌型效果"的管道中消费。印记与卡质互不干扰——宝石质摧毁不清除印记，印记覆盖（商店购买同一张牌的新印记）不影响卡质。这个系统为结算引擎提供印记效果的固定数值输出，为排序系统提供跑鞋/乌龟的优先级信号，为商店系统提供印记定价和库存逻辑。

## Player Fantasy

**序列锻造师 — 编排时序，锻造利器**

核心时刻：商店里，你的筹码只够买一个印记。你有两张方片——方片 Q（效果值 12）和方片 2（效果值 2）。你选择将短剑放在方片 Q 上：12 伤害 + 2 额外 = 14 点输出。但真正的决定不在于"加多少"——而在于你同时给黑桃 9 加了跑鞋。结算开始，黑桃 9 先跑，你攒下 9 点防御；对手的方片 Q 打过来，被防御吃掉；然后你的方片 Q + 短剑在安全位置结算，14 点伤害直取 HP。牌没变，变的只是印记的编排。

印记系统的玩家幻想是**编排者与锻造者合一**：一方面，你像指挥家一样控制结算序列——跑鞋和乌龟重排时序，重锤在同一结算位摧毁对手的卡牌，时机本身就是武器；另一方面，每一次印记选择都是不可逆的建筑决策——短剑让方片变成尖刀，护盾让黑桃变成铁壁，钱币让草花变成印钞机。印记的永久性意味着错误也有重量：给错牌打错印记，比不买更糟。而印记的覆盖范围（多少张牌有印记）与印记的精准度（哪种印记给哪张牌）的权衡，让构筑决策超越了"买最贵的"——而是买最对的。

这服务了游戏的 Roguelike 构筑幻想：你的牌组不再依赖抽牌运气，而是依赖你在商店里精心锻造的印记网络。当一张 2 点牌因为有短剑印记而变得有价值时，你不是在靠运气——你是在靠建筑。

## Detailed Design

### Core Rules

**1. 印记类型定义**

7 种印记，每种赋予卡牌一个固定结算效果：

| 印记 | 枚举值 | 效果类型 | 效果值 | 结算管道位置 |
|------|--------|---------|--------|-------------|
| 短剑 | `SWORD` | 额外伤害 | +2 | 印记效果阶段 |
| 护盾 | `SHIELD` | 额外防御 | +2 | 印记效果阶段 |
| 心 | `HEART` | 额外回复 | +2 | 印记效果阶段 |
| 钱币 | `COIN` | 额外筹码 | +10 | 印记效果阶段 |
| 重锤 | `HAMMER` | 摧毁对手同位卡牌 | — | 印记效果阶段（优先级最高） |
| 跑鞋 | `RUNNING_SHOES` | 优先结算 | — | 排序阶段 |
| 乌龟 | `TURTLE` | 延后结算 | — | 排序阶段 |

**2. 印记与花色的关系**

印记不受花色限制。任何印记可以附加到任何花色的卡牌上，效果独立于花色效果。一张黑桃 9 + 短剑在结算时同时产生两个效果：黑桃花色效果（+9 防御）和短剑印记效果（+2 伤害）。两种效果独立执行，不互相替代。

印记效果类型固定——短剑永远产生伤害，无论卡牌花色是什么。跨花色组合是合法且有意为之的构筑空间：
- 方片 + 短剑 = 纯输出强化（伤害 + 伤害）
- 黑桃 + 短剑 = 武装盾牌（防御 + 伤害）
- 红桃 + 护盾 = 自保治疗（回复 + 防御）
- 草花 + 心 = 经济续航（筹码 + 回复）

**3. 印记在结算管道中的位置**

```
阶段 1: 弹出基础数值 (effect_value)
阶段 2: 印记效果 (stamp effects)
阶段 3: 卡质效果 (quality effects)
阶段 4: 牌型效果 (hand type multiplier)
阶段 5: 计算并执行 (apply to game state)
阶段 6: 摧毁检查 (gem quality destroy roll)
```

印记效果在阶段 2 触发。数值型印记（短剑/护盾/心/钱币）将固定加成加入当前累积值。重锤在预结算阶段（阶段 2 之前）执行全局扫描并标记无效化目标。跑鞋/乌龟没有阶段 2 效果——它们的作用完全在排序阶段完成。

**4. 印记与牌型倍率的交互**

牌型倍率应用于（基础值 + 印记加成 + 卡质加成）的累积总和。印记加成被牌型倍率放大。

```
final_value = (base_effect_value + stamp_bonus + quality_bonus) × hand_type_multiplier
```

示例：方片 7 + 短剑，对子牌型（×2）：
- 基础：7 伤害
- 短剑：+2 = 9
- 牌型 ×2：9 × 2 = **18 伤害**

**5. 重锤机制**

重锤是唯一具有对抗性效果的印记：
- **效果**：将对手同一结算位的卡牌标记为"已无效化"
- **持续时间**：仅当回合。卡牌在弃牌堆后恢复全部功能
- **无效化范围**：被无效化的卡牌跳过阶段 2-6（不触发印记、卡质、牌型效果，不执行任何花色效果，不做摧毁检查）
- **无效化时机**：在交替结算开始前，执行全局重锤扫描（预结算阶段）。扫描所有结算位，标记所有重锤目标。然后在交替结算中，已标记的卡牌跳过阶段 2-6
- **双方重锤**：如果同一结算位双方都有重锤印记，双方卡牌互相无效化，均不产生任何效果（扫描阶段同时标记）
- **无目标**：如果对手手牌数少于该结算位（如玩家有 5 张牌而对手只有 2 张），重锤无目标，效果浪费

**6. 跑鞋与乌龟机制**

跑鞋和乌龟控制卡牌的结算顺序，不产生数值效果：

- **跑鞋**：该卡牌排在该方手牌的最前面（最先结算）
- **乌龟**：该卡牌排在该方手牌的最后面（最后结算）
- **排序范围**：仅影响本方手牌的内部排序，不影响交替结算模式（A1→B1→A2→B2 不变）
- **多张跑鞋子顺序**：如果玩家有 2+ 张跑鞋卡牌，保持玩家手动排序时的相对顺序（跑鞋不改变跑鞋之间的相对位置）
- **跑鞋+乌龟混排**：排序规则为 跑鞋组（保持相对顺序）→ 普通牌组（保持相对顺序）→ 乌龟组（保持相对顺序）
- **先后手交互**：先手方的跑鞋卡牌在结算时仍先于后手方的跑鞋卡牌（因为 A1 在 B1 之前）

**7. 爆牌时印记效果**

当一方爆牌时，该方所有卡牌的花色效果无效——印记效果同样无效。印记在爆牌状态下不触发任何效果，包括数值加成和重锤的无效化效果。跑鞋/乌龟的排序效果不受影响（排序在爆牌检测之前完成）。

**8. 印记持久性与变更**

- 印记通过商店购买附加到卡牌实例上
- 印记在对手之间持续存在（每回合被抽到时都会触发）
- 印记只能通过商店购买新的印记来覆盖——没有"移除印记"的机制
- 覆盖时旧印记完全丢弃，不可恢复
- 同一张卡牌同一时刻只能有一个印记

### States and Transitions

印记本身不驱动状态机。印记的效果体现在结算管道的状态变化中：

```
[无印记] ── 商店购买 ──→ [有印记]
                              │
                              ├── 结算中: 印记效果触发
                              │       │
                              │       ├── 数值型: 加成已加入累积值
                              │       ├── 重锤: 对手卡牌已无效化
                              │       └── 跑鞋/乌龟: 排序已完成（结算前）
                              │
                              └── 商店覆盖: 旧印记丢弃 → [有新印记]
```

### Interactions with Other Systems

| 系统 | 方向 | 数据流 | 触发时机 |
|------|------|--------|---------|
| 卡牌数据模型 | 双向 | 读取/写入 `stamp` 字段 | 商店购买时写入，结算时读取 |
| 结算引擎 | 出 | 印记加成值、无效化标记 | 结算管道阶段 2 |
| 卡牌排序系统 | 出 | `stamp` 是否为 RUNNING_SHOES/TURTLE | 排序阶段（结算前） |
| 商店系统 | 入 | 商店购买请求，写入 `stamp` | 商店阶段 |
| AI 对手系统 | 出 | 生成时随机分配 `stamp` | 新对手开始时 |
| 战斗状态系统 | 间接（经结算引擎） | 印记加成后的伤害/防御/回复值 | 结算管道阶段 5 |
| 筹码经济系统 | 间接（经结算引擎） | 钱币印记的额外筹码 | 结算管道阶段 5 |

## Formulas

### 1. 印记加成查找 (stamp_bonus_lookup)

`stamp_bonus = lookup(stamp_type)`

| stamp_type | bonus_value | effect_type |
|------------|------------|-------------|
| SWORD | 2 | DAMAGE |
| SHIELD | 2 | DEFENSE |
| HEART | 2 | HEAL |
| COIN | 10 | CHIPS |
| HAMMER | — | NULLIFY_TARGET |
| RUNNING_SHOES | — | SORT_FIRST |
| TURTLE | — | SORT_LAST |
| null | 0 | NONE |

**Variables:**
| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| stamp_type | — | 枚举 or null | 8 种取值 | 卡牌实例的印记类型 |

**Output Range**: 数值型 [0, 10]。非数值型印记没有 bonus_value。
**Example**: stamp_type=SWORD → bonus_value=2, effect_type=DAMAGE

### 2. 印记在双轨结算中的贡献

> **注意**：本系统曾定义单一 `final_card_value` 公式，已在卡质系统 GDD 中被拆分为双轨模型。印记加成现在分别被 `combat_effect` 和 `chip_output` 两个公式消费。

印记加成通过两个独立流进入结算引擎：

**战斗效果流 (`combat_effect`)**：
```
combat_effect = (effect_value + stamp_combat_bonus + gem_quality_bonus) × hand_type_multiplier
```
- `stamp_combat_bonus`: SWORD=+2(伤害), SHIELD=+2(防御), HEART=+2(回复), 其他=0
- 完整公式定义见 `card-quality-system.md`，由结算引擎执行

**筹码收益流 (`chip_output`)**：
```
chip_output = (chip_value_base + metal_chip_bonus + gem_chip_bonus + stamp_coin_bonus) × hand_type_multiplier
```
- `stamp_coin_bonus`: COIN=+10(筹码), 其他=0
- 完整公式定义见 `card-quality-system.md`，由结算引擎执行

**Example**: 方片 7 + SWORD + RUBY quality_level II，PAIR 牌型（×2）
- stamp_combat_bonus=2, gem_quality_bonus=4
- combat_effect = (7 + 2 + 4) × 2 = **26 伤害**
- chip_output = (0 + 0 + 0 + 0) × 2 = **0 筹码**（方片牌无筹码产出，无 COIN 印记）

**注意**：黑桃的防御值不受牌型倍率修正后的 combat_effect 影响 — 防御值在回合结束时清零。

### 3. 排序优先级键 (stamp_sort_key)

The `stamp_sort_key` formula is defined as:

```
sort_key = lookup(stamp_type)
```

| stamp_type | sort_key | 说明 |
|------------|----------|------|
| RUNNING_SHOES | 0 | 最先结算 |
| null / 其他数值型 | 1 | 默认位置 |
| TURTLE | 2 | 最后结算 |
| HAMMER | 1 | 重锤不改变排序位置（与数值型相同） |

排序使用稳定排序：按 sort_key 升序排列，相同 sort_key 内保持玩家手动排序的相对顺序。

**Variables:**
| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| stamp_type | — | 枚举 or null | 8 种取值 | 卡牌实例的印记类型 |

**Output Range**: {0, 1, 2}
**Example**: 手牌 [方片7+乌龟, 黑桃9+跑鞋, 红桃J, 草花K+跑鞋, 方片2+重锤]
- sort_keys: [2, 0, 1, 0, 1]
- 稳定排序后: [黑桃9+跑鞋(0), 草花K+跑鞋(0), 红桃J(1), 方片2+重锤(1), 方片7+乌龟(2)]

## Edge Cases

- **如果印记加成与牌型倍率放大了筹码产出**: 钱币（+10 筹码）和祖母绿卡质（+15/20/25 筹码）的加成也会被牌型倍率放大。一个同花（×手牌数）的草花手牌配上钱币印记会产生大量筹码。这是设计意图——经济型构筑通过牌型倍率获得爆发性收益，但依赖同花牌型的低概率触发。

- **如果重锤预结算扫描发现对手卡牌已被爆牌无效化**: 如果一方爆牌，该方所有卡牌已无效。对手的重锤扫描仍会执行，但目标已是无效状态。重锤在此情况下是幂等的——对已无效卡牌再次标记无效不产生额外效果，也不产生额外惩罚。重锤卡牌自身的花色效果仍正常执行。

- **如果重锤无目标时该卡牌仍有花色效果**: 重锤只是"浪费了"无效化能力，但卡牌本身仍正常执行花色效果（阶段 1-6）。一张方片 7 + 重锤在面对只有 2 张牌的对手（而重锤在第 5 位）时，仍然造成 7 点伤害（加印记以外的其他加成）。

- **如果双方重锤在同一结算位互相无效化**: 双方卡牌均不产生任何效果。卡牌进入弃牌堆后在下一回合恢复正常。这是对称的——没有先后手优势。

- **如果所有手牌都有跑鞋（或都有乌龟）**: 稳定排序中所有卡牌 sort_key 相同，排序结果等于玩家的手动顺序。跑鞋/乌龟没有任何净效果。印记槽位被浪费但行为良好。

- **如果重锤卡牌被宝石质摧毁检查移除品质**: 不适用——被重锤无效化的卡牌跳过阶段 6（摧毁检查），其宝石质在当回合受到保护。重锤卡牌自身的摧毁检查不受重锤影响——如果重锤卡牌本身有宝石质且摧毁检查失败，品质被移除但印记保留。

- **如果爆牌方的跑鞋/乌龟排序效果**: 排序发生在爆牌检测之前。跑鞋/乌龟的排序效果不受爆牌影响。但排序后该方所有卡牌的花色效果和印记效果均被爆牌无效化——排序完成但所有后续阶段被跳过。

- **如果分牌后一张分出的手牌爆牌，该手牌上的印记**: 印记跟随卡牌，不跟随手牌。爆牌的那一手牌中的所有卡牌印记无效。另一手牌中的卡牌印记正常触发。

- **如果商店刷出与玩家已有完全相同印记的卡牌**: 商店应灰显该选项并提示"已拥有相同牌"。如果只有印记相同但卡质不同，允许购买并显示差异对比（来自卡牌数据模型边用例）。

- **如果玩家在同一张牌上同时拥有印记和卡质**: 允许。印记和卡质是独立的两个字段。宝石质摧毁不清除印记；印记覆盖不改变卡质。两者在结算管道的不同阶段各自触发。

- **如果牌型检测与重锤无效化的交互**: 牌型检测在结算前执行（读取手牌组成，不受结算状态影响）。被重锤无效化的卡牌仍计入牌型检测——它的倍率被检测到但不执行（被无效化的卡牌跳过执行阶段）。同手牌中未被无效化的其他卡牌仍享受牌型倍率。

- **如果 AI 牌组中重锤超过上限**: 生成后约束检查移除最后添加的超限重锤并重新随机，直到满足 ≤3 的约束。这是卡牌数据模型已定义的行为。

## Tuning Knobs

| 调参点 | 类型 | 默认值 | 安全范围 | 影响什么 |
|--------|------|--------|----------|---------|
| `stamp_sword_bonus` | int | 2 | 1 ~ 10 | 短剑额外伤害。调高增强攻击型构筑，调低使攻击型依赖牌型倍率 |
| `stamp_shield_bonus` | int | 2 | 1 ~ 10 | 护盾额外防御。调高增强防御型构筑，黑桃防御仅用于吸收伤害，不产生反击 |
| `stamp_heart_bonus` | int | 2 | 1 ~ 10 | 心额外回复。调高增强生存型构筑，调低使 HP 管理更紧张 |
| `stamp_coin_bonus` | int | 10 | 5 ~ 50 | 钱币额外筹码。调高加速经济循环（更多商店购买），调低使经济型构筑依赖牌型倍率 |
| `hammer_max_per_ai` | int | 3 | 0 ~ 10 | AI 牌组重锤上限。owned by card-data-model，本系统消费。调高增加 AI 对抗性，调低降低玩家被无效化的挫败感 |
| `stamps_max_per_ai` | int | 30 | 0 ~ 52 | AI 牌组总印记上限。owned by card-data-model，本系统消费。调高增加 AI 复杂度和难度，调低简化 AI 行为 |

**注**：商店印记定价（重锤 300、跑鞋/乌龟 150、其他 100）是商店系统的调参点，不属于本系统。重锤无效化是全有或全无效果——没有"部分无效化"的中间值，因此没有对应的数值调参点。跑鞋/乌龟的效果是结构性（排序优先级），没有数值可调。

**调参交互注意事项**：
- `stamp_sword_bonus` 和 `stamp_shield_bonus` 应保持相同值（当前均为 2）。如果攻击加成明显高于防御加成，攻击型构筑会系统性优于防御型
- 所有数值加成被牌型倍率放大——调高 stamp_bonus 在高倍率牌型下的效果是非线性的（+3 × 7 = +21，而 +2 × 7 = +14）。50% 的基础值增加在高倍率下产生 50% 以上的实际差异
- `stamp_coin_bonus` 通过牌型倍率放大后会显著影响经济节奏——如果经济型构筑过强，优先降低此值或考虑筹码加成不受牌型倍率影响（需要修改 `chip_output` 公式）

## Visual/Audio Requirements

**印记图标**: 每种印记需要独特的图标，显示在卡牌面上。图标应简洁可辨，即使在卡牌缩小显示时也能识别。印记与花色的协同（如 SWORD + DIAMONDS）可通过微妙的发光高亮指示。

**结算触发反馈**:
- 数值型印记触发时：对应的战斗效果应短暂高亮（伤害数字闪红、防御数字闪蓝等），并在数字旁显示印记图标，让玩家识别加成来源
- 重锤无效化：被无效化的卡牌应有明确的视觉标记（如灰色蒙版 + 碎裂动画），无效化瞬间应有打击感音效
- 跑鞋/乌龟：排序后的手牌应有位置编号标识，让玩家确认排序结果

**协同指示**: 当印记与花色匹配（SWORD+方片、SHIELD+黑桃、HEART+红桃、COIN+草花）时，印记图标应有微弱的协同光效。不匹配时不显示光效但仍正常工作。这是教学辅助，不是功能限制。

## UI Requirements

**卡牌展示**: 卡牌组件必须显示印记图标（如果有），位置固定（如左上角或花色图标旁）。空印记位不显示。

**商店印记购买**: 商店的印记选择界面应显示 7 种印记类型，每种标注效果描述和价格。选中印记后高亮可应用的卡牌。已有相同印记的卡牌应灰显或标注"已拥有"。

**结算预览**: 结算前（排序确认后），应显示每张卡的预期效果值，包含印记加成的增量。让玩家在确认前看到"SWORD +2"的具体影响。

**手牌排序**: 拖拽排序时，跑鞋/乌龟卡牌应有视觉提示（如排序方向箭头），表明它们会移动到前排/后排。排序确认后的最终顺序应清晰标注结算位编号（1, 2, 3...）。

> **📌 UX Flag — 印记系统**: This system has UI requirements. In Phase 4 (Pre-Production), run `/ux-design` to create a UX spec for each screen or HUD element this system contributes to **before** writing epics.

## Acceptance Criteria

### 核心规则

**AC-01: 印记加成查找完整性**
GIVEN 7 种印记类型 + null
WHEN 查询 stamp_bonus_lookup
THEN SWORD→(+2, DAMAGE), SHIELD→(+2, DEFENSE), HEART→(+2, HEAL), COIN→(+10, CHIPS), HAMMER→(—, NULLIFY), RUNNING_SHOES→(—, SORT_FIRST), TURTLE→(—, SORT_LAST), null→(0, NONE)。

**AC-02: 印记不受花色限制**
GIVEN 一张黑桃 9 + SWORD 印记
WHEN 结算执行
THEN 黑桃花色效果产生 +9 防御（add_defense），SWORD 印记产生 +2 伤害（apply_damage），两个效果独立执行。

GIVEN 一张红桃 7 + SHIELD 印记
WHEN 结算执行
THEN 红桃花色效果回复 7 HP，SHIELD 印记增加 +2 防御。

**AC-03: 印记在双轨结算中的贡献**
GIVEN 方片 7 + SWORD + RUBY quality_level II，PAIR 牌型（×2）
WHEN 计算 combat_effect
THEN (7 + 2 + 4) × 2 = 26 伤害。

GIVEN 草花 4 + COIN，无卡质，无牌型
WHEN 计算 chip_output
THEN (4 + 10 + 0 + 0) × 1.0 = 14 筹码。

**AC-04: 防御也受牌型倍率影响**
GIVEN 玩家打出黑桃 5 + SHIELD 印记，PAIR 牌型（×2）
WHEN 计算 combat_effect 并结算
THEN SHIELD 加成随牌型倍率放大：defense = (5+2)×2 = 14。结算后 defense 清零。

### 重锤机制

**AC-05: 重锤预结算扫描**
GIVEN 玩家手牌 [D7+HAMMER(pos2), S9(pos1)], AI 手牌 [HQ(pos1), C5(pos2)]
WHEN 预结算扫描执行
THEN AI 的 C5（pos2）被标记为已无效化，AI 的 C5 跳过阶段 2-6。

**AC-06: 双方重锤互相无效化**
GIVEN 玩家 pos3=DQ+HAMMER, AI pos3=HJ+HAMMER
WHEN 预结算扫描执行
THEN 双方 pos3 卡牌均被标记为已无效化，均不产生任何效果。

**AC-07: 重锤无目标**
GIVEN 玩家手牌 5 张（HAMMER 在 pos4），AI 手牌 2 张
WHEN 预结算扫描执行
THEN pos4 无 AI 卡牌，HAMMER 效果浪费，但玩家 pos4 卡牌仍正常执行花色效果。

**AC-08: 被无效化卡牌的印记不触发**
GIVEN AI pos2=SK+SWORD 被 HAMMER 无效化
WHEN 结算执行到 pos2
THEN SK 的黑桃防御效果不触发，SWORD 的 +2 伤害也不触发。

**AC-09: 被无效化卡牌仍计入牌型检测**
GIVEN 玩家手牌 [D7+HAMMER, D7]（对子），AI pos2 有 HAMMER 无效化第一个 D7
WHEN 牌型检测执行
THEN 对子仍被检测到（两张 D7 均在手牌中），未被无效化的 D7 获得 ×2 倍率，被无效化的 D7 不执行。

### 排序机制

**AC-10: 跑鞋/乌龟排序**
GIVEN 手牌 [D7+TURTLE, S9+RUNNING_SHOES, HJ, CK+RUNNING_SHOES, D2+HAMMER]
WHEN 应用 stamp_sort_key 排序
THEN 结果：[S9+RUNNING_SHOES(0), CK+RUNNING_SHOES(0), HJ(1), D2+HAMMER(1), D7+TURTLE(2)]。S9 在 CK 之前（保持手动子顺序）。

**AC-11: 全手牌跑鞋无净效果**
GIVEN 手牌 5 张均有 RUNNING_SHOES
WHEN 应用排序
THEN 结果等于玩家手动顺序（稳定排序，所有 sort_key 相同）。

**AC-12: 排序不影响交替结算模式**
GIVEN 玩家手牌排序后 [S9+RUNNING_SHOES, D7, HJ]，AI 手牌排序后 [DK, C5, S3]
WHEN 交替结算
THEN 顺序仍为 S9→DK→D7→C5→HJ→S3（A1→B1→A2→B2→A3→B3）。

### 爆牌与印记

**AC-13: 爆牌时印记无效**
GIVEN 玩家爆牌（point_total=24），手牌中有 D9+SWORD
WHEN 爆牌检测通过后结算
THEN D9 的花色效果（9 伤害）和 SWORD 印记（+2 伤害）均不触发。

**AC-14: 爆牌时排序仍生效**
GIVEN 玩家手牌中 S9+RUNNING_SHOES 已排序至 pos1
WHEN 爆牌检测通过
THEN 排序结果不变（排序在爆牌检测前完成），但所有后续结算阶段被跳过。

### 持久性与变更

**AC-15: 印记跨对手持续**
GIVEN 玩家在第 2 对手商店给 (PLAYER, DIAMONDS, 7) 购买 SWORD 印记
WHEN 进入第 3 对手的新回合，D7 被抽到
THEN SWORD 印记仍有效，+2 伤害正常触发。

**AC-16: 商店覆盖印记**
GIVEN (PLAYER, SPADES, 9) 当前 stamp=SHIELD
WHEN 商店购买 COIN 印记并应用到该卡
THEN stamp=COIN（旧 SHIELD 完全丢弃，不可恢复）。

**AC-17: 印记与卡质独立**
GIVEN (PLAYER, DIAMONDS, Q) stamp=SWORD, quality=RUBY, quality_level=II
WHEN RUBY 品质被摧毁检查移除
THEN quality=null, quality_level=III, stamp=SWORD（不变）。

GIVEN (PLAYER, HEARTS, J) stamp=HEART, quality=GOLD, quality_level=III
WHEN 商店购买新 stamp=SHIELD
THEN stamp=SHIELD, quality=GOLD, quality_level=III（卡质不受影响）。

### 公式

**AC-F1: combat_effect 覆盖所有场景**
GIVEN base=15(A), stamp=+2(SWORD), quality=+5(RUBY I), multiplier=×2(21 点)
WHEN 计算 combat_effect
THEN (15+2+5)×2 = 44 伤害。

GIVEN base=2(2号), stamp=+2(SWORD), quality=0(null), multiplier=×7(三七)
WHEN 计算 combat_effect
THEN (2+2+0)×7 = 28 伤害。

**AC-F2: stamp_sort_key 边界**
GIVEN 所有 7 种 stamp_type + null
WHEN 查询 sort_key
THEN RUNNING_SHOES→0, {SWORD,SHIELD,HEART,COIN,HAMMER,null}→1, TURTLE→2。
