# 边池系统 (Side Pool System)

> **Status**: In Design
> **Author**: user + agents
> **Last Updated**: 2026-04-25
> **Implements Pillar**: Economy — side betting, risk/reward, casino edge

## Overview

边池系统管理两种可选的副注机制——7边池和赌场战争——为每回合的 21 点对战提供额外的筹码风险/回报层。系统在发牌前（回合 Phase 2）接受玩家的三档下注（10/20/50 筹码），在发牌后根据特定条件结算回报：7边池检测可见的 3 张牌中有多少张 7 并按倍率赔付，赌场战争比较双方 4 张牌中的最大点数归属。边池采用简单倍率模型（无累积池），每次下注独立结算。边池的期望净收益为负值（赌场优势），使其成为筹码消耗口而非收入源——玩家选择边池不是因为数学上有利，而是因为边注带来的额外紧张感和偶尔回报的情绪高潮。在数据层面，系统管理每回合的下注选择和倍率结算；在接口层面，它通过筹码经济系统的 `spend_chips()` 和 `add_chips()` 执行所有资金流转。

## Player Fantasy

**庄家赌性 — 精算之外的直觉之舞**

核心时刻：筹码余额 212，对手 2 的回合开始。边池提示出现。你知道期望回报是负的——你已经算过了。但牌还没发，你心中的赌徒已经苏醒。50 筹码滑入 7 边池。牌发下——方片 7 闪过你的眼角。一张。第二张？对手的明牌……不是。你输了这注。但你笑了——庄家偶尔也赌一把，因为心跳是算不出来的。

边池系统的玩家幻想是**精算师的放纵**——不是"我要赢了"的掌控感，而是"我明知不利还是忍不住"的人性弱点。整个游戏中，你是精算在先的庄家：构建经济引擎、优化结算顺序、在商店做投资组合再平衡。但边池是唯一允许你放下精算、拥抱直觉的机制。你在信息到达之前下注，在概率不利时押注，在确定性游戏中按下"不确定"的按钮。这个幻想衔接了筹码经济的"投资回报"——筹码经济说"我的筹码在工作"，边池说"我的筹码在冒险"。二者不矛盾，它们共同构成了一个完整的赌场人格：庄家精算，赌徒下注，同一个人。

## Detailed Design

### Core Rules

**1. 系统性质**

边池系统管理两种可选副注——7边池和赌场战争——在每回合发牌前接受下注，发牌后立即结算。系统是无状态的：不追踪跨回合的累积金额，每次下注和结算在当前回合内完成。所有资金流转通过筹码经济系统的 `spend_chips()` 和 `add_chips()` 执行。

**2. 边池在回合流程中的位置**

```
Phase 2a: 7边池下注 — 玩家选择下注或跳过
Phase 2b: 赌场战争下注 — 玩家选择下注或跳过
Phase 1:  发牌 — 双方各 2 张
Phase 2c: 7边池结算 — 检查 3 张可见牌中的 7
Phase 2d: 赌场战争结算 — 比较 4 张牌中的最高点数
Phase 3: 保险...
```

下注在发牌前，结算在发牌后立即执行。两个边池可以同时下注（独立金额）。

**3. 下注额度**

两种边池共享相同的下注档位：

| 档位 | 下注金额 |
|------|---------|
| 小注 | 10 筹码 |
| 中注 | 20 筹码 |
| 大注 | 50 筹码 |

玩家可对 7 边池和赌场战争分别选择不同档位，或跳过任一/两者。

**4. 7边池 (7-Side Pool)**

**SP7-1** — 下注时机：发牌前。结算时机：发牌后立即。

**SP7-2** — 检查范围：3 张可见牌 = 玩家 2 张手牌 + AI 1 张明牌。

**SP7-3** — 统计这 3 张牌中 `rank == 7` 的数量，按以下赔率结算：

| 7 的数量 | 概率 | 赔率 | 10注回报 | 20注回报 | 50注回报 |
|---------|------|------|---------|---------|---------|
| 0 张 | 78.5% | ×0（全损） | 0 | 0 | 0 |
| 1 张 | 19.9% | ×2 | 20 | 40 | 100 |
| 2 张 | 1.5% | ×5 | 50 | 100 | 250 |
| 3 张 | 0.035% | ×15 | 150 | 300 | 750 |

**SP7-4** — 下注时调用 `spend_chips(PLAYER, bet, SIDE_POOL_BET)`。结算时若有回报，调用 `add_chips(PLAYER, payout, SIDE_POOL_RETURN)`。0 张 7 时无 `add_chips()` 调用。

**5. 赌场战争 (Casino War)**

**CW-1** — 下注时机：发牌前。结算时机：发牌后，7 边池结算之后。

**CW-2** — 比较范围：4 张牌 = 玩家 2 张 + AI 2 张。

**CW-3** — 找出每方的最高点数牌（rank 比较，A 最高，2 最低）。比较双方最高牌的 rank：
- 玩家最高 > AI 最高：玩家赢，获得 ×2 下注额
- 玩家最高 < AI 最高：玩家输，下注全损
- 玩家最高 = AI 最高（平局）：玩家输，下注全损

**CW-4** — 赔率表：

| 结果 | 概率 | 赔率 | 10注回报 | 20注回报 | 50注回报 |
|------|------|------|---------|---------|---------|
| 玩家赢 | 44.8% | ×2 | 20 | 40 | 100 |
| AI 赢 | 44.8% | ×0 | 0 | 0 | 0 |
| 平局 | 10.3% | ×0 | 0 | 0 | 0 |

**CW-5** — 下注和结算的资金流转同 SP7-4。

**6. AI 行为**

AI 不参与边池——无下注、无结算、无资金交互。AI 的手牌仅作为赌场战争的判定依据。`add_chips(AI)` 和 `spend_chips(AI)` 均为空操作。

**7. 爆牌/特殊玩法交互**

边池在发牌后立即结算，先于保险、分牌、双倍下注、要牌/停牌。边池结果不受后续任何阶段影响：
- 爆牌不影响边池（已结算完毕）
- 即时胜利（SPADE_BLACKJACK）不影响边池（已结算完毕）
- 分牌不影响边池（3 张可见牌基于初始发牌，不因分牌改变）
- 双倍下注不影响边池

### States and Transitions

边池系统无跨回合状态。每个回合的下注-结算周期独立：

```
[Phase 2a] → 玩家选择 7 边池档位 (10/20/50/跳过)
                 │ spend_chips() if bet > 0
                 ▼
[Phase 2b] → 玩家选择赌场战争档位 (10/20/50/跳过)
                 │ spend_chips() if bet > 0
                 ▼
[Phase 1]  → 发牌 (各 2 张)
                 │
                 ▼
[Phase 2c] → 7 边池结算
                 │ count_sevens(visible_3_cards)
                 │ add_chips() if payout > 0
                 ▼
[Phase 2d] → 赌场战争结算
                 │ compare max_rank(player_2, ai_2)
                 │ add_chips() if player wins
                 ▼
[Phase 3]  → 保险...
```

### Interactions with Other Systems

| 系统 | 边池接收 | 边池提供 | 触发时机 |
|------|---------|---------|---------|
| 卡牌数据模型 (#1) | 读取 `rank`（7 边池检测 rank==7；赌场战争比较 rank 大小） | 无 | Phase 2c/2d |
| 筹码经济 (#10) | `spend_chips()` 下注扣款成功/失败 | `add_chips()` 结算回报 | Phase 2a/2b (下注)、2c/2d (结算) |
| 回合管理 (#13) | 阶段调度时机（Phase 2 位置） | 边池结果（赢/输/跳过）用于 UI 反馈 | 回合阶段切换 |
| 特殊玩法 (#8) | 无 | 无 | 无直接交互（边池先于特殊玩法结算） |
| 牌桌 UI (#15) | 无 | 下注状态、结算结果用于显示 | 实时 |

## Formulas

### 1. 7边池赔付 (sp7_payout)

The `sp7_payout` formula is defined as:

`sp7_gross_return = min(floor(bet_sp7 × M7[seven_count]), CHIP_CAP)`

**Variables:**
| Variable | Symbol | Type | Range | Description |
|----------|--------|------|-------|-------------|
| bet_sp7 | B₇ | int | {0, 10, 20, 50} | 7 边池下注金额；0 = 跳过 |
| seven_count | k | int | {0, 1, 2, 3} | 3 张可见牌中 rank==7 的数量 |
| M7 | — | int[4] | {0, 2, 5, 15} | 赔率查找表，按 seven_count 索引 |
| CHIP_CAP | — | int | 999 | 已注册常量；余额硬上限 |
| sp7_gross_return | — | int | [0, 750] | 通过 add_chips() 返还玩家的金额 |

**Output Range:** [0, 750]。最大值为 50×15=750，远低于 CHIP_CAP。
**Example:** bet=20, 2 张 7 → `floor(20 × 5) = 100`。`add_chips(PLAYER, 100, SIDE_POOL_RETURN)`。

### 2. 赌场战争赔付 (cw_payout)

The `cw_payout` formula is defined as:

`cw_gross_return = min(floor(bet_cw × M_cw[outcome]), CHIP_CAP)`

outcome 判定：`max_rank_player` vs `max_rank_ai`（A=14 最高，2=2 最低）
- WIN: `max_rank_player > max_rank_ai`
- LOSE: `max_rank_player < max_rank_ai`
- TIE: `max_rank_player == max_rank_ai` → 按输处理

**Variables:**
| Variable | Symbol | Type | Range | Description |
|----------|--------|------|-------|-------------|
| bet_cw | B_cw | int | {0, 10, 20, 50} | 赌场战争下注金额；0 = 跳过 |
| max_rank_player | R_p | int | {2..14} | 玩家 2 张牌中的最高 rank |
| max_rank_ai | R_a | int | {2..14} | AI 2 张牌中的最高 rank |
| outcome | — | enum | {WIN, LOSE, TIE} | 比较结果 |
| M_cw | — | int[3] | {2, 0, 0} | 赔率查找表（WIN=2, LOSE=0, TIE=0） |
| cw_gross_return | — | int | [0, 100] | 通过 add_chips() 返还的金额 |

**Output Range:** [0, 100]。最大值为 50×2=100。
**Example:** bet=50, 玩家[A,3] vs AI[K,9] → max_p=14(A) > max_a=13(K) → WIN → `floor(50 × 2) = 100`。

### 3. 期望值与庄家优势

**7 边池 EV:** `sp7_ev = Σ P7[k] × M7[k]` = 0.7852×0 + 0.1991×2 + 0.0153×5 + 0.00035×15 = **0.480**
**7 边池庄家优势:** `sp7_house_edge = 1 - sp7_ev` = **52.0%**

**赌场战争 EV:** `cw_ev = P_win × 2` = 0.4483 × 2 = **0.897**
**赌场战争庄家优势:** `cw_house_edge = 1 - cw_ev` = **10.3%**（完全来自平局输的规则）

**每场预期筹码消耗（每回合下注 20 筹码，8 回合）:**
- 7 边池: 20 × 0.52 × 8 = **83 筹码/场**
- 赌场战争: 20 × 0.103 × 8 = **17 筹码/场**

## Edge Cases

- **If 玩家余额不足以下注任一边池（余额 < 10）**: 两个边池自动跳过，无 `spend_chips()` 调用。余额不变。若玩家为后手，50 筹码补偿在边池阶段之前触发（回合管理 `on_round_start`），可能使余额恢复到可下注范围。

- **If 玩家仅能负担一个边池的下注（余额 = 15）**: 玩家可下注 10 筹码于 7 边池，但赌场战争所有档位不可用。Phase 2a 的 `spend_chips()` 成功后余额变为 5，Phase 2b 的所有选项灰显。

- **If 玩家同时下注两个边池后余额归零，且 AI 明牌为 A**: 边池下注已消耗全部筹码（如 SP7=10 + CW=20，余额从 30→0）。保险窗口 `can_afford(30) = false`，仅 HP 支付路径可用。这是战略取舍：重注边池可能锁定保险的筹码支付路径。

- **If 7 边池最大赔付（×15 on 50 = 750）将余额推超上限**: `add_chips(PLAYER, 750, SIDE_POOL_RETURN)` 钳制到 999。例：余额 930（下注后）+ 750 = 1680 → 钳制为 999，实际增益 69。上限钳制是筹码经济系统的全局行为，边池不例外。在高余额时，7 边池的期望值进一步下降。

- **If 两个边池在同一回合都赔付，累积触发上限**: SP7 先结算，CW 后结算。每次 `add_chips()` 独立检查上限。若 SP7 赔付已将余额推至 999，CW 赔付被完全钳制，实际增益为 0。

- **If 赌场战争双方最高 rank 相同（平局）**: 按输处理，下注全损。例：玩家[K,3] vs AI[K,7] → 双方最高=K → 平局 → 玩家输。整个 10.3% 庄家优势来自此规则。

- **If 赌场战争 rank 比较使用的是 rank 而非 bj_value**: CW-3 使用 rank 排序（A=14 最高，2=2 最低），不是 21 点计算值（A=11/1）。实现时 `max_rank = max(card_a.rank_value, card_b.rank_value)` 其中 rank_value 映射：2→2, ..., 10→10, J→11, Q→12, K→13, A→14。

- **If SPADE_BLACKJACK 即时胜利跳过全部结算**: 边池在 Phase 2c/2d 已结算完毕，先于 Phase 0a 即时胜利判定。边池结果不受影响——赢了就是赢了，输了就是输了。

- **If 玩家分牌后边池已结算**: 边池基于初始发牌的 3 张可见牌（Phase 2c），先于分牌检查（Phase 4）。分牌不影响边池结果。

- **If 玩家爆牌但已赢得边池**: 爆牌标记影响结算引擎（Phase 1-6 跳过，chip_output=0），但边池赔付已在 Phase 2c/2d 入账。爆牌不追溯扣除边池收入。

- **If 筹码经济 GDD 中的边池金额与边池系统不一致**: 筹码经济 GDD 已同步更新为 10/20/50。如未来任一 GDD 修改下注金额，需同步更新另一 GDD 并更新注册表常量。

## Tuning Knobs

| 旋钮 | 类型 | 默认值 | 安全范围 | 影响什么 |
|------|------|--------|----------|---------|
| `sp7_multiplier_0` | int | 0 | 0 | 0 张 7 的赔率。不应修改（0 = 全损） |
| `sp7_multiplier_1` | int | 2 | 1–5 | 1 张 7 的赔率。对 EV 影响最大（19.9% 命中率）。+1 此值 = +19.9% EV |
| `sp7_multiplier_2` | int | 5 | 2–20 | 2 张 7 的赔率。中等影响（1.5% 命中率）。+1 = +1.5% EV |
| `sp7_multiplier_3` | int | 15 | 5–100 | 3 张 7 的赔率（头奖）。EV 影响极小（0.035% 命中率）。主要控制头奖兴奋感 |
| `cw_win_multiplier` | int | 2 | 1–3 | 赌场战争赢的赔率。设为 3 时 EV>1（玩家有利！）。设为 1 时庄家优势 55% |
| `cw_tie_result` | enum | LOSE | {LOSE, PUSH} | 平局处理。LOSE = 10.3% 庄家优势。PUSH = 0% 庄家优势（公平赌局） |
| `bet_tier_small` | int | 10 | 5–50 | 小注金额。不影响庄家优势（EV 按单位计算），控制绝对筹码流速 |
| `bet_tier_medium` | int | 20 | 10–100 | 中注金额。三档应保持递增且总和不超过初始筹码 |
| `bet_tier_large` | int | 50 | 20–200 | 大注金额。最大注不应超过初始筹码的 50%（100×50%=50） |

**旋钮交互：**
- `sp7_multiplier_1` 是平衡的主要杠杆。它是 19.9% 命中率的乘数——从 2 调到 3 会将庄家优势从 52% 降至约 32%。如 playtest 表明 52% 过于激进，优先调此旋钮。
- `cw_win_multiplier` 和 `cw_tie_result` 联动：若平局=PUSH（公平），`cw_win_multiplier` 必须 <2 才能产生庄家优势。
- 三个下注档位不影响庄家优势，但影响玩家的筹码流速和心理感受。大额下注使输赢更有戏剧性。

## Acceptance Criteria

**AC-01** — **GIVEN** 玩家下注 20 筹码于 7 边池，**WHEN** 3 张可见牌中 0 张 rank==7，**THEN** 无 `add_chips()` 调用，20 筹码已消费。

**AC-02** — **GIVEN** 玩家下注 10 筹码于 7 边池，**WHEN** 恰好 1 张可见牌 rank==7，**THEN** `add_chips(PLAYER, 20, SIDE_POOL_RETURN)` 被调用。

**AC-03** — **GIVEN** 玩家下注 20 筹码于 7 边池，**WHEN** 恰好 2 张可见牌 rank==7，**THEN** `add_chips(PLAYER, 100, SIDE_POOL_RETURN)` 被调用。

**AC-04** — **GIVEN** 玩家下注 50 筹码于 7 边池，**WHEN** 3 张可见牌全部 rank==7，**THEN** `add_chips(PLAYER, 750, SIDE_POOL_RETURN)` 被调用。

**AC-05** — **GIVEN** 玩家手牌 [7♥, K♠]，AI 明牌 7♦，**WHEN** 7 边池计数，**THEN** seven_count=2（K 的 rank 不为 7）。

**AC-06** — **GIVEN** 玩家下注 50 筹码于赌场战争，玩家 [A,3] vs AI [K,9]（max_rank 14 > 13），**WHEN** 赌场战争结算，**THEN** `add_chips(PLAYER, 100, SIDE_POOL_RETURN)` 被调用。

**AC-07** — **GIVEN** 玩家下注 20 筹码于赌场战争，玩家 [4,9] vs AI [5,Q]（max_rank 9 < 12），**WHEN** 赌场战争结算，**THEN** 无 `add_chips()` 调用。

**AC-08** — **GIVEN** 玩家下注 10 筹码于赌场战争，双方最高 rank 均为 K（平局），**WHEN** 赌场战争结算，**THEN** 平局按输处理，无 `add_chips()` 调用。

**AC-09** — **GIVEN** 玩家 [A♠, 3♥] vs AI [10♣, 9♦]，**WHEN** 赌场战争计算 max_rank，**THEN** max_rank_player=14（rank 值，非 bj_value=11）。

**AC-10** — **GIVEN** 玩家余额 80，**WHEN** Phase 2a 下注 SP7=20 成功（余额→60），Phase 2b 下注 CW=50 成功（余额→10），**THEN** 两个边池独立生效。
